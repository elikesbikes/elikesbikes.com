  resources/list.html   Resources landing — card grid of repo sections

  └ tutorials   → /resources/tutorials/

## Content Types & Layouts

### `type: "repo"` pages
Used by all pages under `content/resources/*/`.
Rendered by `layouts/repo/list.html`.

Frontmatter fields:
- `title` — repo/page name
- `description` — one-liner shown under the title
- `github_url` — renders the green "View on GitHub →" button
- `tech` — array of strings rendered as badge pills
- `folders` — array of `{name, description, url}` rendered as a card grid
  - If `url` starts with `http`, opens in new tab with "Open in GitHub ↗"
  - If `url` is a local path (e.g. `/resources/tutorials/docker-compose/`), opens internally with "View details →"

### Homepage (`/`)
Rendered by `layouts/index.html` (custom — overrides Blowfish default).

Layout order:
1. Profile photo (centered, 120px circle) — from `params.author.image` in `languages.en.toml`
2. Name — from `params.author.name`
3. Headline — from `params.author.headline`
4. Bio text — from `content/_index.md` body
5. Social link pills — YouTube, GitHub, Discord, Strava, TrainerRoad (hardcoded in layout, icons via Simple Icons CDN)
6. Recent Project card — driven by `recent_project` frontmatter in `content/_index.md`

Frontmatter fields for `recent_project`:
- `name` — project/folder name
- `description` — one-liner shown below the thumbnail
- `url` — GitHub URL (opens in new tab)
- `thumbnail` — path to a 16:9 image (e.g. `/images/projects/n8n.svg`); optional — card omits image block if missing
- `icons` — array of `{name, url}` using Simple Icons CDN (`https://cdn.simpleicons.org/{icon}/{color}`)

The card renders as a vertical tile: large 16:9 thumbnail on top, then icons + name + GitHub button + description below — matching the style of YouTube video cards.

To update the featured project, edit `content/_index.md`. Add the SVG thumbnail to `static/images/projects/`. Browse icons at https://simpleicons.org.
To add/remove social links, edit `layouts/index.html` directly.

### Resources landing (`/resources/`)
Rendered by `layouts/resources/list.html`.
Iterates `.Sections` to show repo cards with description, tech badges, and links.

### About page (`/about/`)
Rendered by `layouts/about/list.html`.
All content is hardcoded in the layout (profile photo, bio sections, social cards).
The `_index.md` just needs frontmatter — no body content required.

### Videos (`/videos/`)
Rendered by `layouts/videos/list.html`.
Each video is a markdown file with `youtube_id`, `tags`, `description` in frontmatter.
Generated by `scripts/fetch_youtube.ym`.

## Search

Blowfish search requires three things — all in place:
1. `enableSearch = true` in `config/_default/params.toml`
2. `home = ["HTML", "RSS", "JSON"]` in `config/_default/hugo.toml` (generates `/index.json`)
3. `content/search/_index.md` with `layout: "search"` (entry point for the search page)

Search icon in navbar links to `/search/`. Powered by Fuse.js (client-side, no server needed).

## Build & Deploy

```bash
./build.sh          # Downloads Hugo, initializes submodules, builds with --gc --minify
wrangler deploy     # Deploy to Cloudflare Workers (reads wrangler.toml)
```

Output goes to `./public/`. Cloudflare Workers serves the static assets.
Custom 404 handling is configured in `wrangler.toml`.

## Key Conventions

- **Dark theme**: background `#0f1720`, cards `#1a2535`, borders `#2d3748`, accent blue `#7fb3f5`
- **All custom layouts use inline styles** — no external CSS framework beyond Blowfish defaults
- **Folder card grids** use `grid-template-columns: repeat(auto-fit, minmax(260px, 1fr))`
- **Hover effects** on cards use inline `onmouseover`/`onmouseout` to toggle `borderColor`
- **Blowfish theme files** are not in the repo — loaded via Hugo modules at build time. Do not try to edit files in `themes/blowfish/` directly
- **`_index.md` files** are section list pages in Hugo, rendered by `list.html` layouts (not `single.html`)
- The `layouts/_default/list.html` wraps content in `{{ if .Content }}` and pages list in `{{ if .Pages }}` to avoid empty elements
